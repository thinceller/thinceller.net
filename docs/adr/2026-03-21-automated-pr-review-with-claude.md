# Automated PR Review with Claude Code Actions

## Status

accepted

## Context

プルリクエストのコードレビューは重要だが、以下の課題があった：

- 個人プロジェクトのため、セルフレビューに限界がある
- レビュー観点の漏れ（セキュリティ、パフォーマンス、一貫性等）
- 依存関係更新PRの影響度判定が手動で時間がかかる
- コードレビューの品質が一定でない

## Decision

**Claude Code Actions** を使用した自動PRレビューワークフローを導入する。

2つのワークフローを実装：

### 1. 汎用PRレビュー (`.github/workflows/review-pr.yml`)

6つの観点で並列にサブエージェントを起動し、包括的なレビューを実施：

1. **Code Quality**: 可読性、命名規則、複雑さ、DRY原則
2. **Bug & Regression Detection**: ロジックエラー、エッジケース、既存機能への影響
3. **Security**: XSS、インジェクション攻撃、認証/認可の問題
4. **Performance**: 不要な再レンダリング、N+1問題、バンドルサイズ影響
5. **Consistency**: AGENTS.mdに記載されたコーディング規約との整合性
6. **TypeScript Type Safety**: `any`の使用、型アサーション、型ガードの欠落

各サブエージェントは以下を返す：
- ファイルパス
- 行番号
- 重要度（high/medium/low）
- 説明

レビュー結果はGitHubコメントとして投稿され、インラインコメントも付与される。

### 2. 依存関係更新PR向けレビュー (`.github/workflows/review-deps.yml`)

Renovateボットが作成した依存関係更新PRを自動レビュー：

- 変更内容の影響度判定（breaking changes, 機能追加、バグ修正等）
- 低リスクの更新は自動承認（approve）
- 高リスクの更新は詳細なレビューコメントを投稿

### トリガー条件

- **汎用レビュー**: Renovate以外が作成したPR、draft状態でない
- **依存関係レビュー**: Renovateが作成したPR

### Considered Options

| オプション | 特徴 | 評価 |
|------------|------|------|
| **Claude Code Actions** | AIによる多角的レビュー、コンテキスト理解が優れる | ★★★ |
| **CodeRabbit** | AIレビューツール、GitHub連携 | ★★ |
| **SonarQube** | 静的解析ツール、カバレッジ計測 | ★★ |
| **手動レビューのみ** | 自動化なし | ★ |

### 選択理由

1. **多角的なレビュー**: 6つの異なる観点で並列レビュー
2. **コンテキスト理解**: AIがプロジェクト全体の文脈を理解してレビュー
3. **AGENTS.md連携**: プロジェクト固有の規約に基づいたレビュー
4. **並列実行**: サブエージェントによる高速なレビュー
5. **GitHub統合**: PRコメントとインラインコメントの自動投稿
6. **依存関係更新の効率化**: Renovate PRの自動トリアージ

## Consequences

### より容易になること

- **レビューの網羅性向上**: 6つの観点で漏れなくチェック
- **セキュリティリスクの早期発見**: 専門的なセキュリティレビュー
- **依存関係更新の効率化**: 低リスク更新の自動承認
- **コード品質の一貫性**: プロジェクト規約への準拠確認
- **学習効果**: AIレビューから改善点を学べる

### より困難になること

  - 対策: 重要度highのみを強調表示

### 技術的詳細

- **認証**: `CLAUDE_CODE_OAUTH_TOKEN` シークレットを使用
- **権限**: `contents: read`, `pull-requests: write`, `id-token: write`
- **実行環境**: `ubuntu-latest`
- **Claude Code Action**: `anthropics/claude-code-action@v1`
