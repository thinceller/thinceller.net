# Automated PR Review with Claude Code Actions

## Status

accepted

## Context

個人開発かつAIエージェント開発において、コードレビューが人間に属人化していた。

課題：
- **セルフレビューの限界**: 個人開発のため客観的な視点が不足
- **レビュー観点の漏れ**: セキュリティ、パフォーマンス、一貫性などの多角的な観点でのチェックが不十分
- **依存関係更新PRの負荷**: Renovateボットが作成する大量の依存関係更新PRの影響度判定が手動で時間がかかる
- **AIエージェント開発の制約**: AIエージェントが作成したPRを人間がレビューする必要があるが、レビューコメントを見て自動修正できると効率的

## Decision

**Claude Code Actions** を使用した自動PRレビューワークフローを導入する。

### ワークフロー構成

**1. 汎用PRレビュー** (`.github/workflows/review-pr.yml`)

6つの観点で並列にサブエージェントを起動し、包括的なレビューを実施：
- Code Quality: 可読性、命名規則、複雑さ、DRY原則
- Bug & Regression Detection: ロジックエラー、エッジケース、既存機能への影響
- Security: XSS、インジェクション攻撃、認証/認可の問題
- Performance: 不要な再レンダリング、N+1問題、バンドルサイズ影響
- Consistency: CLAUDE.mdに記載されたコーディング規約との整合性
- TypeScript Type Safety: `any`の使用、型アサーション、型ガード

各サブエージェントはファイルパス、行番号、重要度（high/medium/low）、説明を返す。レビュー結果はGitHubコメントとして投稿され、インラインコメントも付与される。

**2. 依存関係更新PR向けレビュー** (`.github/workflows/review-deps.yml`)

Renovateボットが作成した依存関係更新PRを自動レビュー：
- 変更内容の影響度判定（breaking changes、機能追加、バグ修正等）
- 低リスクの更新は自動承認（approve）
  - 注: ブランチ保護により、最終的なマージには人間の承認が必要
- 高リスクの更新は詳細なレビューコメントを投稿

### Considered Options

**1. Claude Code Actions（採用）**
- AIによる多角的レビュー
- プロジェクトコンテキストを理解してレビュー
- CLAUDE.mdとの連携

**2. CodeRabbit**
- AIレビューツール、GitHub連携
- ただしプロジェクト固有の規約への対応が不明

**3. 手動レビューのみ**
- 属人化が続く
- レビュー観点の漏れが残る

### 選択理由

- **多角的なレビュー**: 6つの異なる観点で並列レビュー、漏れを防ぐ
- **コンテキスト理解**: AIがプロジェクト全体の文脈（CLAUDE.md等）を理解してレビュー
- **AIエージェントとの協調**: AIエージェントがレビューコメントを見て自動で修正できる
- **依存関係更新の効率化**: Renovate PRの自動トリアージで作業負荷を削減

## Consequences

**より容易になったこと:**
- CIでレビューが走ることで、AIエージェントがレビューコメントを見て自動で修正できるようになった
- 6つの観点で漏れなくチェックされるようになった
- セキュリティリスクや型安全性の問題を専門的な視点で早期発見
- 依存関係更新PRのトリアージが自動化された
- コード品質の一貫性が向上（CLAUDE.mdに記載されたプロジェクト規約への準拠確認）

**より困難になったこと:**
- Claude API使用料が発生する
- False Positive（誤検知）が発生する可能性（重要度レベルで優先度判断）
- レビューコメント数が多くなる可能性
- セキュリティ考慮事項:
  - プロンプトインジェクション攻撃: 悪意あるPRのdiffにレビュー出力を操作する指示を埋め込むリスク
  - 緩和策: ワークフローは`pull-requests: write`権限のみ（コード変更権限なし）、人間による最終承認が必須、Claudeのシステムプロンプトによるガードレール
