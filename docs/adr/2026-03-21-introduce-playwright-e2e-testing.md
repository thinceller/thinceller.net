# Introduce Playwright for E2E Testing

## Status

accepted

## Context

プロジェクトにはユニットテストやコンポーネントテストが存在せず、リグレッションの検出が困難だった。特に以下の課題があった：

- デプロイ前の動作確認が手動に依存
- ページ表示、ナビゲーション、RSSフィードなどの基本機能の自動検証がない
- リファクタリングや依存関係更新時の影響範囲の把握が困難
- CI/CDパイプラインでの品質担保が不十分

## Decision

**Playwright** をE2Eテストフレームワークとして導入する。

### 実装内容

1. **テストフレームワーク**: `@playwright/test` v1.58.2
2. **対象ブラウザ**: Chromiumのみ（デスクトップChrome）
3. **テストポート**: 3001（開発サーバーの3000と分離）
4. **テストスイート構成**:
   - スモークテスト: 主要ページの表示確認（/, /about, /blog, /blog/[slug]）
   - RSSフィード検証: rss.xml, atom.xmlの妥当性確認
   - ナビゲーションテスト: ページ間遷移の動作確認

5. **CI統合**: GitHub Actions ワークフロー (`.github/workflows/e2e.yml`)
   - プルリクエスト時に自動実行
   - リトライ機能: CIでは2回、ローカルでは0回
   - アーティファクト保存: テスト失敗時のスクリーンショット・動画

6. **動的記事取得**:
   - テスト対象の記事slug をハードコードせず、`_posts/` ディレクトリを走査して`publishedTime`が最新の記事を自動取得
   - 新記事追加時もテストコードの変更不要

### Considered Options

| オプション | 特徴 | 評価 |
|------------|------|------|
| **Playwright** | Microsoft製、高速、複数ブラウザ対応、優れたDX | ★★★ |
| **Cypress** | 人気が高い、豊富なエコシステム | ★★ |
| **Puppeteer** | Google製、Chromiumのみ | ★ |
| **テストなし** | E2Eテストを導入しない | ★ |

### 選択理由

1. **パフォーマンス**: Playwrightは並列実行とブラウザ起動が高速
2. **開発者体験**: TypeScript統合、Auto-waiting、豊富なアサーション
3. **CI統合**: GitHub Actions公式サポート、キャッシング効率が良い
4. **将来の拡張性**: 複数ブラウザ対応が容易（現在はChromiumのみだが、将来Firefox/Safariも追加可能）
5. **Next.jsとの相性**: Next.jsプロジェクトでの採用実績が豊富
6. **保守性**: Microsoft製で長期的なサポートが期待できる

## Consequences

### より容易になること

- **リグレッション検出**: 自動テストによる早期発見
- **CI/CD品質担保**: プルリクエスト時の自動検証
- **リファクタリングの安全性**: 既存機能の動作保証
- **デプロイ前の信頼性**: 本番環境相当のビルドでテスト実行
- **ドキュメント効果**: テストコードが期待動作の明示的な仕様となる

### より困難になること

- **CIの実行時間増加**: E2Eテスト実行により数分追加
  - 対策: 並列実行とキャッシングで最小化（現在のテスト実行時間: <!-- TODO: 実測値を記載 -->）
- **テストの保守コスト**: UIや機能変更時にテストも更新が必要
- **フレイキーテスト対策**: 非決定的な失敗への対応が必要
  - 対策: Playwrightの自動待機機能とリトライメカニズムで緩和
- **依存関係の増加**: `@playwright/test` とブラウザバイナリ（約200MB）
  - 対策: CI環境ではキャッシング

### 技術的詳細

- **設定ファイル**: `playwright.config.ts`
- **テストディレクトリ**: `e2e/`
- **スクリプト**:
  - `pnpm test:e2e`: ヘッドレスモード実行
  - `pnpm test:e2e:ui`: UIモード（デバッグ用）
- **webServer設定**: テスト前に `next build && next start -p 3001` を自動実行
