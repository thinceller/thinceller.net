# Migrate to Biome for Code Quality Tooling

## Status

accepted

## Context

プロジェクトではコード品質管理のために複数のツールを使用していた。コードフォーマット、リンティング、インポート整理など、それぞれ個別のツールが必要だった。

課題:
- 複数ツールの設定ファイル管理
- ツール間の設定の整合性維持
- 実行速度

## Decision

コード品質管理ツールを **Biome** に統一する。

### Considered Options

1. **Biome**: ESLint/Prettierの代替となる統合ツール
2. **ESLint + Prettier**: 従来の組み合わせを維持
3. **その他の統合ツール**: <!-- TODO: 検討した他のツールがあれば記載 -->

### 選択理由

<!-- TODO: 以下の理由を確認・詳細化してください -->
- **パフォーマンス**: Rust製で高速
- **統合性**: フォーマット、リンティング、インポート整理を1つのツールで実現
- **設定の簡素化**: 単一の設定ファイル (biome.json)
- **TypeScript/JSXのサポート**: Next.js/Reactプロジェクトに適合
- **Tailwind CSS v4のサポート**: CSS parserでtailwindDirectivesをサポート

### 設定内容

- 行幅: 80文字
- インデント: 2スペース
- JSX引用符: ダブルクォート
- JavaScript引用符: シングルクォート
- 末尾カンマ: 常につける
- セミコロン: 常につける
- `noImgElement` ルール: Next.js Imageコンポーネント使用を強制
- OGP画像生成用ファイルでは `noImgElement` を除外

## Consequences

### より容易になること

- **高速なリンティング・フォーマット**: Rust製により従来より高速
- **設定管理の簡素化**: 単一の設定ファイルで完結
- **一貫性**: 単一ツールによる統一されたコード品質管理
- **メンテナンスコスト削減**: 依存関係が減少
