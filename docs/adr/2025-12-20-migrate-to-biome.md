# Migrate to Biome for Code Quality Tooling

## Status

accepted

## Context

プロジェクトではESLintとPrettierを別々のツールとして使用しており、それぞれにプラグインを個別に導入・管理する必要があった。

課題：
- **設定ファイルの分散**: `.eslintrc.js`, `.prettierrc`, `.prettierignore` など複数ファイルの管理
- **プラグインの個別管理**: ESLintプラグイン、Prettierプラグインをそれぞれ更新
- **実行速度**: JavaScript製ツールのため、リンティング・フォーマットに時間がかかる
- **フィードバックループ**: コード変更から問題検出までが遅い

## Decision

コード品質管理ツールを **Biome** に統一する。

Rust製の統合ツールで、リンティング・フォーマット・インポート整理を単一ツールで提供する。

### Considered Options

**1. Biome**
- Rust製で高速
- リンティングとフォーマットを統合
- 設定ファイルが1つ（`biome.json`）
- Tailwind CSS v4のサポート

**2. ESLint + Prettier（現状維持）**
- 成熟したエコシステム
- 豊富なプラグイン
- ただし速度面と管理面の課題は残る

### 選択理由

- **パフォーマンス**: Rust製で既存ツールより大幅に高速
- **統合性**: 単一ツールで完結し、設定の不整合がなくなる
- **保守性**: アップデートが1パッケージで済む
- **開発体験**: 高速なフィードバックループにより開発効率が向上

## Consequences

**より容易になったこと:**
- ツールが統一され、`biome.json` 1つで設定が完結
- パッケージ更新が1つで済むため保守が楽
- Rust製の高速動作により、フィードバックループが速くなった
- CI実行時間も短縮

**より困難になったこと:**
- ESLint固有のプラグインエコシステムには依存できなくなった
- チーム開発の場合、メンバーのエディタ拡張を切り替える必要がある
